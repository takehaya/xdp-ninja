\section{Limitations}
\label{sec:limitations}

\kunai is a DSL specialized for packet filtering rather than a
general-purpose eBPF language. Stateful filtering, deep packet
inspection, and aggregation are out of scope, and P4 defines only
header layout and parser transitions, leaving pipeline mechanisms such
as actions and tables outside the language.

We establish verifier acceptance empirically, without a formal
guarantee~\cite{vishwanathan2022tristate}. Acceptance for an arbitrary
\kunai expression on an arbitrary kernel version remains future work,
and a filter that uses \texttt{bpf\_loop} has a kernel-version lower
bound. The generated bytecode is also only as correct as the protocol
definition, since the p4c parse check verifies P4 syntax rather than
agreement with the relevant RFC. Coverage is still limited, and Geneve
option TLVs and the tc-stripped outer VLAN tag are not yet exposed as
filter targets (\S\ref{sec:eval-accept}).
