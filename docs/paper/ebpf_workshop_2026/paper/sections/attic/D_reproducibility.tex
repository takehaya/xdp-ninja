\section{Reproducibility}
\label{app:reproducibility}

The full \kunai+\xdpninja source tree, evaluation harness, and raw
CSV data backing every claim in \S\ref{sec:eval} are available at
\texttt{[URL anonymized for review]}.

\paragraph{Build.} \texttt{make build} produces the
\texttt{xdp-ninja} binary; the \kunai compilation pipeline is a Go
library with no \texttt{clang}/\texttt{llc}/\texttt{bpftool}
dependency at runtime.

\paragraph{Verifier matrix (\S\ref{sec:codegen-verifier}).}
\texttt{make test-bpf} drives \texttt{vimto}-managed QEMU images of
Linux 6.1, 6.6, 6.12, 6.18, and 7.0, loading the canonical Filter Set
F1--F10 against both the XDP and tc host adapters. The matrix is
also regenerated by the \texttt{bpf\_load\_test} GitHub Actions
workflow on every commit.

\paragraph{Codegen overhead B1/B2 (\S\ref{sec:eval-codegen}).}
\texttt{benchmark/microbench/run.sh} produces
\texttt{benchmark/results/b1\_insns.csv} (raw verifier-slot counts)
and \texttt{benchmark/results/b2\_runtime.csv}
(\texttt{BPF\_PROG\_TEST\_RUN}-based per-packet runtime, 10$^6$
iterations).

\paragraph{Throughput B3 and visibility V1 (\S\ref{sec:eval-throughput}, \S\ref{sec:eval-visibility}).}
\texttt{benchmark/pipelines/*\_run.sh} orchestrates T-Rex profiles
in \texttt{benchmark/trex/} and writes per-experiment results to
\texttt{benchmark/results/}: F1 single-source match-rate sweep at
22.8\,Mpps offered is split across
\texttt{b3\_throughput\_perf.csv} (perf event array baseline) and
\texttt{b3\_throughput\_ringbuf\_v1.csv} (\texttt{bpf\_ringbuf}
transport, the 873\,kpps and 1.43$\times$ figures);
\texttt{r5\_multi\_source.csv} carries the 64-RX-queue multi-source
comparison; \texttt{r7\_raw\_dump.csv} the
\texttt{-{}-raw-dump} mode; and
\texttt{visibility\_matrix.csv} the five production-XDP V1 rows
behind Table~\ref{tab:visibility-matrix}. Testbed: Intel Xeon
Platinum 8362 (32C/64T) DUT and Generator, Intel E810-C
100\,GbE NIC, Linux 6.18.
