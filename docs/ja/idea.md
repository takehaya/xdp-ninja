ベースのアイディアメモ
XDPの処理のアイディア。
従来パケットを処理するときにxdpcapなどで処理後を取得することはできるが、処理前を取得する方がなかった。
そこで処理前を取得するアイディアを思いついたので書いておく
専用のフィルタを作ってロードするアイディアがある。

具体的にはメインハンドラで受けた時に、その間にもう一個tailcallを入れておく。
Ingress -> Filterd（アイディア部分） -> mainの処理...->TX/PASS/... などのイメージだ。
で、そこには制御用のebpf mapがあって、有効化されたら特定のvlan/ip/teidごとにフィルタがかけれるようにしておく。
そしてperf lingbuf経由で特定のパケットユーザー空間に集めるイメージ。

で、さらにExtenedなアイディアがある。
今書いたのは、そもそものフィルタを書く場所が書けるだろというアイディアだけど、動的にフィルタを生成して、perf lingbufに送りつけてユーザーランドアプリケーションで取り出せないかな？というアイディア。そうすると性能低下を抑えながら処理するっていうのができて嬉しいし、ユーザーランドアプリを適切に使ってる限りは実行時にだけにフィルタが掛かるはず。

例えば、ここのxdpdumpは任意のところに仕掛けてpacketdumpができるはず。ここに任意のところに仕掛ける際にbpfのプログラムを動的に生成したらいいんじゃないか？というのがアイディアの肝
